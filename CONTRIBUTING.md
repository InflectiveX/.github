# Contributing to Inflectivex Projects

Thank you for your interest in contributing to an Inflectivex project.  
This document provides guidelines and expectations for contributions.

## Contribution Process

1. **Review the Repository Documentation**  
   Before contributing, review the README, architecture notes, and existing issues.

2. **Open an Issue**  
   For bugs, enhancements, or design discussions, create an issue and provide clear context, steps to reproduce (if applicable), and your proposed approach.

3. **Discuss Before Implementing**  
   For larger changes, wait for project maintainers to review your proposal to ensure alignment with project direction.

4. **Create a Feature Branch**  
   Use a descriptive branch name such as:  
   `feature/<name>`  
   `fix/<name>`  
   `hotfix/<name>`

5. **Submit a Pull Request**  
   Include a clear summary, the motivation for the change, and reference any related issues.  
   Pull requests must pass automated checks and receive approval from maintainers.

## Coding Standards

- Follow the language/framework-specific style guides used in the repository.
- Write clear, maintainable, and tested code.
- Include documentation updates when necessary.

## Code of Conduct

All contributors are expected to act professionally and respectfully.  
For concerns, contact the project maintainers directly.
