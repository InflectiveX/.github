# Security Policy

Inflectivex takes security seriously and appreciates responsible disclosure of vulnerabilities.

## Reporting a Vulnerability

If you believe you have identified a security issue, please report it privately.  
Do not create a public issue.

Contact the Inflectivex security team at:

**security@inflectivex.com**  

Include:

- Description of the potential vulnerability  
- Steps to reproduce  
- Impact assessment  
- Any relevant logs or proof-of-concept materials  

You will receive an acknowledgment within a reasonable timeframe.

## Disclosure

We request that you do not publicly disclose the issue until:

- We have reviewed the report  
- A fix has been developed  
- A coordinated release has been completed  

Inflectivex does not offer formal bug bounty programs at this time unless specified per project.
